# Project Name

## Purpose
This repository follows a standardized structure for experiments, projects, or research work.
It is designed to separate code, documentation, outputs, tests, and optional add-ons for clarity and maintainability.

## Folder Structure
- docs/                 Documentation, specs, architecture
- src/                  Source code
- src/importer/         Backup parsing or core logic
- src/ui/               Streamlit or other UI code
- src/utils/            Shared helper functions
- outputs/              Generated files, logs, weekly outputs
- outputs/logs/         Logs
- outputs/weekly_lists/ Weekly follow-up lists or exports
- tests/                Unit tests or sample backup tests
- tests/sample_backups/ Example input files
- configs/              Configuration files (optional)
- data/                 Sample datasets (optional)
- notebooks/            Jupyter notebooks (optional)
- scripts/              Helper or CLI scripts (optional)

## Notes
- Folder-specific README.md files should be maintained manually as the project evolves.
- The repository is ready for GitHub Desktop or any Git workflow.
